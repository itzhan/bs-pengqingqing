题目：基于Spring Boot+Vue的非遗技艺“师徒档案”传承管理系统设计与实现
研究（设计）目的与任务：
研究目的：
本课题面向非物质文化遗产“口传心授、缺乏留痕、难以规模化管理”的核心痛点，构建一套基于 Spring Boot + Vue 的“师徒档案”传承管理系统，实现“档案标准化建档—线上拜师与关系绑定—教学任务与课程管理—作品上传与点评—成长记录与可视化—统计报表与资料库”的全流程数字化。系统区分师傅、徒弟与管理员三类角色，通过RBAC权限、结构化数据模型与可追溯的操作审计，既提升传承过程的可视化与规范化程度，也为文化数字化与非遗保护提供可复制的工程范式与数据基础。
研究任务：
1.需求分析：完成传承组织、师徒个体与管理部门的场景调研与用例梳理。
2.系统设计：完成总体架构、模块划分、接口规范、权限模型与数据库E-R设计，明确师傅/徒弟/关系、非遗项目、教学任务、作品、成长记录、活动课程、公告与审计日志等域模型。
3.功能实现：
1）师傅端：个人档案完善、徒弟申请审核、教学任务发布与材料上传、作品在线点评、成长轨迹查看；
2）徒弟端：注册登录与拜师申请、参与教学任务、作品提交与修改、查看点评与成长报告、活动课程参与；
3）管理员端：账号与师徒关系审核、非遗项目库与技艺类别维护、档案与作品资料库管理、活动课程发布、评价内容监管、统计报表与系统配置。
4.交互与体验：完成低门槛的操作流程与人文化界面，形成高可用原型并迭代优化。
5.测试与优化：覆盖功能/接口/性能/安全用例，重点验证文件上传与访问控制、关系变更一致性、统计口径与报表正确性。
6.文档与论文：提交设计说明书、数据库说明、接口文档、测试报告、部署手册与用户手册，完成论文撰写与修改。
基本要求（应进行的调查研究或实验实习、研究方法、应掌握的原始资料（数据）、参考文献及其设计 技术要求、注意事项等，基本要求应符合《郑州西亚斯学院本科毕业论文（设计）工作管理条例》有关 规定）：
1.设计和分析的技术要求：熟练掌握 SSM 框架和相关技术，包括 Spring 、SpringMVC 、MyBatis、 MySQL 等，在分析和设计过程中，鼓励使用面向对象技术和 UML 建模工具。
2.功能要求：能够实现管理系统的核心功能，必须包含但不限于：师傅用户的登录注册、完善个人档案（技艺类别、从艺经历、代表作品等）、审核与管理徒弟申请、发布教学任务与学习计划、在线指导与点评徒弟作品、查看弟子成长记录功能；徒弟用户的登录注册、申请拜师与解除绑定、查看师傅信息与教学内容、上传学习心得与作品、参与课程与活动、查看师傅点评与个人成长记录功能；管理员用户的师傅与徒弟账号审核与管理、非遗项目与技艺类别管理、师徒关系管理、档案与作品资料库维护、活动与课程信息发布、评价内容监管、统计分析与报表导出、系统公告与权限设置功能
3.用户体验要求：注重用户体验，设计出简洁、易用的界面，提高用户的使用体验和满意度。同时， 需要考虑响应速度和页面加载性能，确保用户能够快速访问和使用系统。
4.数据安全要求：确保用户数据的安全性，采取合适的数据加密和存储技术，防止用户数据泄露。 同时，需要遵守相关法律法规和政策要求，保护用户隐私。
5.可扩展性和可维护性要求：设计平台时需要考虑平台的可扩展性和可维护性， 以便于未来对平台 进行升级和扩展。同时，需要编写清晰的开发文档和使用手册，方便后续的维护和升级。
6.权限管理与认证要求：设计系统的权限管理与认证功能，确保只有经过授权的用户才能访问和使 用系统。同时，需要考虑如何保证权限管理的安全性和准确性。
7.性能要求：确保系统的性能满足需求，包括平均用户请求响应速度、并发处理能力等，满足中小 企业常规使用。
8.测试与优化要求：对文档管理系统进行全面的测试，包括功能测试、性能测试等。确保系统的功 能和性能满足需求，并优化系统的性能和响应速度。
9.论文要求：按照学校规定，在毕业设计工作的时间节点内完成毕业设计和论文撰写工作。论文撰 写符合学校规范,各项毕业设计材料符合学校规范，且完备。
10.参考文献符合下列要求：合理引用文献数据、观点、原理等材料，但不能抄袭。查询近 5 年与论文研究内容相关的文献资料，主要包括国内外期刊论文、行业报告、研究报告和著作等。查阅参考文献资料，并标注在论文中不少于 15 个，至少有 2 篇外文文献。
11.答辩准备充分：毕设需要进行充分的答辩准备，论文查重符合学校规定后，准备详细的演示 PPT 和项目报告等材料。能够清晰地展示系统的功能和成果，回答评审专家的问题。

技术要求:
	•	后端：Spring Boot 构建 REST API；MyBatis 或 MyBatis-Plus 完成ORM/分页/事务；Spring Security + JWT 完成认证与授权
	•	前端：Vue + 组件库（如 Element Plus）实现路由化与组件化；Axios 对接 API；完善表单校验与状态反馈；支持移动端自适应。
	•	数据库：MySQL 设计师傅、徒弟、关系、非遗项目、教学任务、作品（含多媒体元数据）、成长记录、活动课程、公告、审计日志等表；关键查询建立复合索引；对文件仅存储路径与元数据。
	•	可视化与报表：ECharts 展示师徒规模、活跃度、课程参与、作品数量与技艺分布等指标
	•	工程化与部署：提供 .env 样例与初始化 SQL；日志分级、操作审计、备份恢复与故障应急方案齐备；本地与服务器环境可运行演示。

	•	测试阶段需确保主要功能模块通过功能、性能与安全测试；
